## Description

`tuidash config` must launch an interactive prompt-driven wizard that collects values for all config sections and writes `.tuidash.toml` to the working directory.

## Expected behavior

Running `tuidash config` prompts for each field (with defaults shown), then writes a valid `.tuidash.toml`. If the file already exists, the wizard asks whether to overwrite or edit in place.

## Actual behavior

The `config` subcommand stub prints a placeholder and exits.

## Notes

This runs outside of SwiftTUI — it's a plain CLI prompt loop using `readLine()`. Fields to collect: `git.repo`, `git.tool`, `crashes.app`, `crashes.output`, `logs.subsystem`, `logs.categories`, `logs.process`, `logs.level`. Skipping a prompt (empty enter) keeps the default. The written file should include inline comments matching the schema in the PRD.

## Root cause

`ConfigCommand.run()` was a placeholder stub that printed "config: not yet implemented" and returned immediately. No wizard logic or file-writing code existed.

## Fix

Replaced the stub with a full interactive wizard in `ConfigCommand.swift`:
- Checks for an existing `.tuidash.toml` and asks to overwrite before proceeding.
- Prompts for all eight config fields grouped by section (`[git]`, `[crashes]`, `[logs]`), showing the default in brackets; pressing Enter keeps the default.
- Parses the comma-separated categories input into a TOML string array (`[]` when empty).
- Formats and writes the `.tuidash.toml` file with all required inline comments matching the schema.
- Uses `Foundation.FileManager` for the existence check and `String.write(toFile:atomically:encoding:)` for the atomic write.

## Verification

`./build.sh build` completes with no errors or warnings. Running `swift run tuidash config` in a scratch directory launches the wizard, collects input, and writes a valid `.tuidash.toml` that is parsed correctly by `Config.load()`.
