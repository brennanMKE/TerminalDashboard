# 0004 — Implement tuidash config interactive setup wizard

| | |
|---|---|
| **Status** | open |
| **Module** | Commands / Config |
| **Platform** | macOS |
| **First seen** | 2026-05-22 |

## Description

`tuidash config` must launch an interactive prompt-driven wizard that collects values for all config sections and writes `.tuidash.toml` to the working directory.

## Expected behavior

Running `tuidash config` prompts for each field (with defaults shown), then writes a valid `.tuidash.toml`. If the file already exists, the wizard asks whether to overwrite or edit in place.

## Actual behavior

The `config` subcommand stub prints a placeholder and exits.

## Notes

This runs outside of SwiftTUI — it's a plain CLI prompt loop using `readLine()`. Fields to collect: `git.repo`, `git.tool`, `crashes.app`, `crashes.output`, `logs.subsystem`, `logs.categories`, `logs.process`, `logs.level`. Skipping a prompt (empty enter) keeps the default. The written file should include inline comments matching the schema in the PRD.
