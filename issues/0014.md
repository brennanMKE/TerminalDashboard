## Description

`AutoCoordinator` subscribes to all three data sources, manages badge counts, drives view switching, and runs the 10-second revert timer when an auto switch occurs.

## Expected behavior

- Subscribes to event streams from `GitDataSource`, `CrashesDataSource`, and `LogsDataSource`.
- On each incoming event: increments the badge count for the event's source view.
- Switching rules:
  - `critical` always triggers a switch to the emitting view.
  - `error` triggers a switch unless the currently visible view already has an active `error` or `critical` event.
  - `info` and `warning` do not trigger a switch — badge only.
- On auto switch: records the current **home view**, shows the new view, starts a 10s countdown.
- After 10s: reverts to the home view, clears the countdown banner.
- If a second significant event fires during countdown: resets the timer to 10s; switches view if higher priority.
- Manually switching a view: that view becomes the new home view, countdown is cancelled, Auto Mode remains active.
- Cmd+Shift+A during countdown: suspends Auto Mode, current view stays pinned, no revert.
- Badge for a view clears when that view becomes visible.
- Exposes `@Published` state: active view, badge counts, auto mode flag, countdown value, banner message.

## Actual behavior

No coordinator exists.

## Notes

The coordinator is a Swift `@MainActor` class (or actor) since it drives UI state. The 10s countdown should update every second so the tab bar can show a live countdown. The home view defaults to whichever view is active at launch.
