## Description

`CrashesDataSource` must watch the two macOS crash log directories for new `.ips` and `.crash` files, match them against a configured app name filter, and emit `DashboardEvent` values.

## Expected behavior

- Watches `~/Library/Logs/DiagnosticReports/` and `/Library/Logs/DiagnosticReports/` using `DispatchSource` filesystem events, falling back to ≤5s polling if unavailable.
- Scans for existing files at launch.
- New file matching app filter (case-insensitive prefix): emits `critical`.
- Any other new file: emits `info`.
- Exposes a sorted list of matched and unmatched reports consumed by `CrashesView`.
- Works when one or both directories are absent.

## Actual behavior

No data source exists.

## Notes

Use `DispatchSource.makeFileSystemObjectSource` for the watch. The app filter is a case-insensitive prefix match against the filename. Expose enough metadata per file (name, date, process, exception type) for the view to render without re-reading every file on each tick.

Since macOS 12 the primary format is `.ips` (two concatenated JSON objects: header + payload). Older `.crash` text files may still appear. The same directory also holds `.hang`, `.spin`, `.cpu_resource`, and other non-crash diagnostics — the data source should include these in the unmatched section rather than ignore them. See [docs/collecting-crash-logs-macos.md](../docs/collecting-crash-logs-macos.md) for the full directory layout and file format details.

## Root cause

No `CrashesDataSource` actor existed; the data source layer had a gap between `GitDataSource` and whatever will serve `CrashesView`.

## Fix

Implemented `CrashesDataSource.swift` containing:

- `CrashReport` — `Sendable` model struct with `filename`, `date`, `matchesFilter`, `fileExtension`.
- `CrashesDataSource` — Swift actor that:
  - Watches `~/Library/Logs/DiagnosticReports/` and `/Library/Logs/DiagnosticReports/` via `DispatchSource.makeFileSystemObjectSource` (`.write` mask on the directory fd).
  - Falls back gracefully when a directory is absent (no crash, just skips source setup and retries on each poll tick).
  - Polls every ≤5 seconds to catch directories that appear after launch and files missed between DispatchSource notifications.
  - Scans existing files at launch (`emitEvents: false`) to populate initial state without triggering events.
  - Monitors extensions: `.ips`, `.crash`, `.hang`, `.spin`, `.cpu_resource`, `.wakeups_resource`, `.diag`, `.panic`.
  - App filter: case-insensitive prefix match against filename; emits `.critical` for matches, `.info` for all other new files.
  - Exposes `matchedReports`, `unmatchedReports` (both sorted newest-first), `appFilter`, `errorMessage`.
- `DispatchSourceWrapper` — `@unchecked Sendable` reference-type wrapper with `nonisolated init` to satisfy Swift 6 `defaultIsolation(MainActor.self)` without importing the non-`Sendable` `DispatchSourceFileSystemObject` into MainActor context.
- `CrashesState` — `@MainActor ObservableObject` that mirrors the `GitState` pattern: owns the actor, exposes `@Published` properties, drives event subscription and periodic sync, and exposes an `onEvent` callback.
