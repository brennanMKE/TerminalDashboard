# 0017 — Suspend and resume data sources on display sleep and wake

| | |
|---|---|
| **Status** | open |
| **Module** | DataSources / Coordinator |
| **Platform** | macOS |
| **First seen** | 2026-05-22 |

## Description

When a MacBook display goes to sleep — either from inactivity, lid close, or manual sleep — the three data sources continue polling and streaming unnecessarily. The tool should detect display sleep and suspend all active data sources, then resume them when the display wakes.

## Expected behavior

- On display sleep: all running data sources are suspended.
  - `GitDataSource` stops its polling loop.
  - `CrashesDataSource` pauses its filesystem watch and polling.
  - `LogsDataSource` stops the `log stream` subprocess.
- On display wake: all suspended data sources resume automatically.
  - Each data source restarts from its last known state.
  - `LogsDataSource` relaunches the `log stream` subprocess.
  - The UI reflects the resumed state within the normal poll interval.
- In auto mode, the `AutoCoordinator` coordinates suspend/resume across all three sources.
- In dedicated mode, the single active data source is suspended and resumed directly.

## Steps to reproduce

1. Run `tuidash` or any subcommand.
2. Let the display sleep (inactivity timeout, close lid, or `pmset sleepnow`).
3. Observe: data sources continue running unnecessarily in the background.

## Expected behavior

All data sources idle while the display is asleep and resume cleanly on wake.

## Actual behavior

Data sources run continuously regardless of display state.

## Notes

Use `IOKit` display sleep/wake notifications via `IORegisterForSystemPower` or the simpler `NSWorkspace.shared.notificationCenter` with `NSWorkspace.screensDidSleepNotification` and `NSWorkspace.screensDidWakeNotification`. The `NSWorkspace` approach is simpler and does not require IOKit directly — prefer it unless finer-grained control is needed.

Each data source needs `suspend()` and `resume()` methods (or equivalent). `GitDataSource` can cancel its polling `Task` and restart it on resume. `CrashesDataSource` can cancel the `DispatchSource` watchers and recreate them on resume. `LogsDataSource` can terminate the `Process` on suspend and relaunch it on resume.

A `SleepWakeMonitor` helper (separate file) should own the notification subscriptions and call into the coordinator or directly into the state objects. It must be `@MainActor` since it drives UI-adjacent state.
