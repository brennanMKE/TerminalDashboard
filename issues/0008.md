## Description

`LogsDataSource` must launch a `log stream` subprocess with the configured subsystem/category/process/level filters, parse the ndjson output line by line, and emit `DashboardEvent` values.

## Expected behavior

- Launches `log stream --style ndjson --subsystem <value>` with optional additional flags.
- Parses each JSON line for `timestamp`, `messageType` (level), `subsystem`, `category`, and `eventMessage`.
- Emits `info` for `debug`/`info`/`notice` messages.
- Emits `warning` for `warning` messages.
- Emits `error` for `error`/`fault`/`critical` messages; debounced to at most one `error` event per 10s.
- Exposes a ring buffer of recent log entries consumed by `LogsView`.
- Subprocess is terminated cleanly on stop/deinit.
- Missing `log` binary or no `subsystem` configured: error state, no events.

## Actual behavior

No data source exists.

## Notes

Use `Process` + `Pipe` and read stdout asynchronously. The ndjson format emits one JSON object per line. The debounce on `error` events prevents the auto coordinator from thrashing views during a burst of errors. The ring buffer size should be at least 1000 entries; the view will truncate to what fits on screen.

## Root cause

No `LogsDataSource` existed. The `DataSources` module had `GitDataSource` and `CrashesDataSource` but nothing for the `log stream` subsystem.

## Fix

Created `/Sources/TerminalDashboard/DataSources/LogsDataSource.swift` containing:

- `LogLevel` enum (`debug`, `info`, `notice`, `warning`, `error`, `fault`) — `Sendable`
- `LogEntry` struct — `Sendable`, `Identifiable`, stores `timestamp` as raw string
- `LogsDataSource` Swift actor — launches `log stream --style ndjson --subsystem <value>` via `Process`+`Pipe`, reads stdout asynchronously line by line using `availableData`, parses each ndjson line with `JSONSerialization`, maps `messageType` to `LogLevel`, maintains a 1000-entry ring buffer, and emits `DashboardEvent` values (`.error`/`.fault` → debounced at 10s, `.warning` → warning, others → info)
- `LogsState` `@MainActor ObservableObject` — mirrors actor state onto `@Published` properties (`entries`, `isPaused`, `filterSummary`, `errorMessage`), buffers new entries when paused and flushes on `resume()`, exposes `onEvent` callback for coordinator use

Missing `log` binary or absent `subsystem` config sets `errorMessage` and emits no events.
