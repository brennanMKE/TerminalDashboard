## Description

`LogsDataSource` must launch a `log stream` subprocess with the configured subsystem/category/process/level filters, parse the ndjson output line by line, and emit `DashboardEvent` values.

## Expected behavior

- Launches `log stream --style ndjson --subsystem <value>` with optional additional flags.
- Parses each JSON line for `timestamp`, `messageType` (level), `subsystem`, `category`, and `eventMessage`.
- Emits `info` for `debug`/`info`/`notice` messages.
- Emits `warning` for `warning` messages.
- Emits `error` for `error`/`fault`/`critical` messages; debounced to at most one `error` event per 10s.
- Exposes a ring buffer of recent log entries consumed by `LogsView`.
- Subprocess is terminated cleanly on stop/deinit.
- Missing `log` binary or no `subsystem` configured: error state, no events.

## Actual behavior

No data source exists.

## Notes

Use `Process` + `Pipe` and read stdout asynchronously. The ndjson format emits one JSON object per line. The debounce on `error` events prevents the auto coordinator from thrashing views during a burst of errors. The ring buffer size should be at least 1000 entries; the view will truncate to what fits on screen.
