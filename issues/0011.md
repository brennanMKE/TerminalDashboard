## Description

`LogsView` renders the live log stream from `LogsDataSource` with color-coded severity levels, a ring-buffer display, and pause/resume support.

## Expected behavior

- Log lines displayed newest at bottom, truncated to terminal width.
- Ring buffer: only the most recent N lines that fit on screen are shown.
- Color scheme:
  - `debug` — dim/gray
  - `info` — default
  - `notice` — cyan
  - `warning` — yellow
  - `error` — red
  - `fault` / `critical` — red, bold
- Footer shows pause/live indicator and active filter summary. No persistent key hints.
- `Space` toggles pause/resume — pausing freezes the displayed buffer; resuming re-attaches.
- `c` clears the buffer.
- "Not configured" prompt shown if no `subsystem` is set.

## Actual behavior

No view exists.

## Notes

The color scheme requires SwiftTUI's `.foregroundColor()` modifier. Verify the available color values against the SwiftTUI API at implementation time — not all SwiftUI colors may be available. The pause state should be a `@State` bool in the view; when paused, the view stops consuming new entries from the data source's ring buffer.
