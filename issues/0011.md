## Description

`LogsView` renders the live log stream from `LogsDataSource` with color-coded severity levels, a ring-buffer display, and pause/resume support.

## Expected behavior

- Log lines displayed newest at bottom, truncated to terminal width.
- Ring buffer: only the most recent N lines that fit on screen are shown.
- Color scheme:
  - `debug` — dim/gray
  - `info` — default
  - `notice` — cyan
  - `warning` — yellow
  - `error` — red
  - `fault` / `critical` — red, bold
- Footer shows pause/live indicator and active filter summary. No persistent key hints.
- `Space` toggles pause/resume — pausing freezes the displayed buffer; resuming re-attaches.
- `c` clears the buffer.
- "Not configured" prompt shown if no `subsystem` is set.

## Actual behavior

No view exists.

## Root cause

`LogsView` had not yet been implemented. `LogsState` also lacked `togglePause()` and `clear()` convenience methods needed by the view.

## Fix

- Created `Sources/TerminalDashboard/Views/LogsView.swift` following the established `GitView`/`CrashesView` pattern:
  - Outer `LogsView` wraps a `GeometryReader` to obtain terminal dimensions.
  - Inner `LogsContentView` computes `visibleCount = terminalHeight - 2` (chrome: footer + command input row) and slices `state.entries.suffix(visibleCount)` to display the ring buffer tail.
  - `@ViewBuilder entryText(_:level:)` applies `.foregroundColor` per severity; `.fault` additionally applies `.bold()`.
  - Footer row shows `PAUSED` (yellow, bold) or `LIVE` (green) alongside `state.filterSummary`.
  - TextField command handler dispatches `p`/space → `state.togglePause()`, `c` → `state.clear()`.
  - "Not configured" message shown when `state.errorMessage` indicates a missing subsystem.
- Added `togglePause()` and `clear()` to `LogsState` in `LogsDataSource.swift`.
