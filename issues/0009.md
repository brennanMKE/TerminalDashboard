## Description

`GitView` is the SwiftTUI view that renders the git working-tree state. It consumes `GitDataSource` and fills the terminal with a header, file list, and status footer.

## Expected behavior

- Header row: repo name, branch, ahead/behind counts (↑ N ↓ N), last-updated timestamp.
- File list: one row per changed file with status flag (M / A / D / ? / R), staging state, and path. Truncates to available terminal height; overflow count in footer.
- Footer: result of last git operation or error message. No persistent key hints.
- Key bindings active when this view is focused:
  - `p` — `git push` with confirmation prompt if branch has diverged.
  - `u` — `git pull` (fast-forward only).
  - `r` — `git pull --rebase`.
  - `o` — launch the configured `tool` command with `{dir}` replaced by repo path (no-op if unconfigured).
- All git operations run as background tasks; view refreshes on completion.
- Non-git directory or missing `git` binary shows a clear error without crashing.

## Actual behavior

No view exists.

## Root cause

The `Views/` directory and `GitView.swift` had not been created yet.

## Fix

Created `Sources/TerminalDashboard/Views/GitView.swift` with:

- **`GitView`** — top-level `ObservableObject`-consuming view; uses `GeometryReader` to measure terminal height and pass it to the content view.
- **`GitContentView`** — pure layout view with header row, file list section, footer row, and a command `TextField`.
- Header shows repo name (bold), branch (cyan), ahead/behind arrow labels (green/yellow), and last-updated timestamp.
- File list uses `ForEach` over `state.files.prefix(visibleCount)` where `visibleCount = terminalHeight − 3 chrome rows`. Overflow count appears in the footer.
- Staged/worktree flag characters are color-coded: staged `M`/`A` green, `D` red, `R`/`C` cyan; worktree `M` yellow, `D` red, `?` gray.
- Footer shows the last operation result or `state.errorMessage` in red on error.
- Command input: a `TextField` with placeholder `p/u/r/o` (or `y/n` when confirming a diverged push). On Return, the first character dispatches the corresponding git operation.
- Git operations (`push`, `pull --ff-only`, `pull --rebase`) run in a `Task { @MainActor }` block via a private `runGit(args:repoPath:)` function using `Process` + `withCheckedContinuation`. stderr is captured and surfaced as the footer error message.
- Tool launch (`o`) uses `/bin/sh -c` fire-and-forget with `{dir}` substitution.
- `GitViewError` private enum satisfies the Swift 6 requirement that `Result`'s `Failure` type conforms to `Error`.

SwiftTUI's `PrimitiveView`, `Node`, and `Control` are all `internal` to the library and cannot be subclassed from outside. Single-keypress bindings therefore use a `TextField` (type command + Return) rather than raw `handleEvent` interception, which is the correct SwiftTUI public-API pattern.
