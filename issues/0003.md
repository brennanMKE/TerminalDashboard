## Description

The tool needs to read a TOML config file so it can run without requiring CLI arguments every time. A `Config` type must parse `.tuidash.toml` and expose typed values for all three data sources. CLI flags must take precedence over the file.

## Expected behavior

Resolution order:
1. CLI flags (highest precedence).
2. `.tuidash.toml` in the working directory.
3. `~/.config/tuidash/config.toml` (global fallback).

Parsed values are available as a `Config` struct consumed by each subcommand.

## Actual behavior

No config file support exists.

## Notes

TOML schema from PRD:
```toml
[git]
repo = "."
tool = "gitup {dir} -t"

[crashes]
app = "MyApp"
output = "./crashes"

[logs]
subsystem = "com.example.MyApp"
categories = ["networking", "auth"]
process = ""
level = "info"
```
Swift does not have a built-in TOML parser. Options: add a TOML package dependency (e.g. `TOMLKit`) or write a minimal parser for this known schema. Evaluate at implementation time. If a required value (`app`, `subsystem`) is missing from both config and flags, the relevant view shows a "not configured" prompt — it does not crash.

## Resolution

Implemented a minimal TOML parser in `Sources/TerminalDashboard/Config.swift` with no new dependencies. The parser handles the known schema: `[git]`, `[crashes]`, and `[logs]` sections with string and string-array values. Inline `#` comments are stripped correctly (respecting quoted strings).

## What was done

- Added `Config`, `GitConfig`, `CrashesConfig`, and `LogsConfig` structs with `Equatable` conformance and all fields as `Optional`.
- Implemented `Config.load(override:)` applying the three-level precedence chain: explicit path → `.tuidash.toml` in CWD → `~/.config/tuidash/config.toml`.
- Parsing is split into a pure `Config.parse(toml:)` method (unit-testable, no I/O) and a thin `Config.parse(contentsOf:)` file-reading wrapper.
- Missing required fields (`app`, `subsystem`) result in `nil` — callers handle the absent value; the parser never throws for missing keys.

## What was not done

Unit test target not added (no `Tests/` directory exists in the package yet); that is tracked separately.
