## Description

The package currently only depends on swift-argument-parser and has no platform constraint. SwiftTUI must be added as a dependency and the package must target macOS 13+ before any view or TUI code can be written.

## Steps to reproduce

1. Open `Package.swift`.
2. Note there is no `platforms` field and no SwiftTUI dependency.

## Expected behavior

`Package.swift` declares `platforms: [.macOS(.v13)]`, adds the SwiftTUI package dependency, and threads it into the `TerminalDashboard` target.

## Actual behavior

No platform constraint and no SwiftTUI dependency present.

## Notes

SwiftTUI repo: https://github.com/rensbreur/SwiftTUI — has historically been pinned by branch (`main`) rather than semver. Verify the current recommended pinning at implementation time. The `defaultIsolation(MainActor.self)` swift setting is already in place in `Package.swift`.

## Root cause

`Package.swift` was scaffolded with only swift-argument-parser and no platform declaration, making it impossible to add SwiftTUI (which requires macOS 13+) or write any TUI views.

## Fix

- Added `platforms: [.macOS(.v13)]` to the `Package` initializer.
- Added `.package(url: "https://github.com/rensbreur/SwiftTUI.git", branch: "main")` to the package-level `dependencies` array.
- Added `.product(name: "SwiftTUI", package: "SwiftTUI")` to the `TerminalDashboard` target's `dependencies` array.
- Kept the existing `defaultIsolation(MainActor.self)` swift setting unchanged.

## Verification

Running `./build.sh build` after the change produced `Build complete! (10.77s)` with all 80 compile/link steps succeeding. SwiftTUI was resolved from the `main` branch and the `tuidash` executable was linked successfully.

## Files changed

- `Package.swift` — added `platforms`, SwiftTUI package dependency, and target product dependency.
- `Package.resolved` — updated by SwiftPM to record the resolved SwiftTUI revision.
