## Description

The auto mode shell requires a persistent tab bar at the top showing all three views with activity badges and an AUTO mode indicator.

## Expected behavior

- One tab per view: `[G] Git`, `[C] Crashes`, `[L] Logs`.
- Active tab is bold/highlighted.
- Inactive tabs show a badge for unseen significant events:
  - `●` — one new event since last viewed.
  - `●●` — two or more events.
  - Badge clears when that view becomes active.
- `AUTO` indicator on the right: bright when Auto Mode is active, dim when suspended.
- Transient countdown banner replaces normal tab bar content during an auto switch: `"Auto → Logs: error detected — returning in 8s"`. Returns to normal after the 10s window elapses or is cancelled.
- Tab bar is a single fixed-height row at the top of the terminal.

## Actual behavior

No tab bar exists.

## Notes

The tab bar should be driven by observable state from `AutoCoordinator` — badge counts, active view, auto mode flag, and the countdown timer value. The countdown display counts down in real time (updating each second).
