## Description

The app entry point is a single `ParsableCommand` stub. It needs to be restructured into a root command with five subcommands — `git`, `crashes`, `logs`, `auto`, and `config` — each in its own file. Running `tuidash` with no subcommand must default to `auto`.

## Expected behavior

- `tuidash auto` (or bare `tuidash`) prints a placeholder and exits cleanly.
- `tuidash git`, `tuidash crashes`, `tuidash logs` each print a placeholder.
- `tuidash config` prints a placeholder.
- `tuidash --help` lists all five subcommands.

## Actual behavior

A single root command with a `print("Hello, world!")` body. No subcommands exist.

## Notes

Use ArgumentParser's `@OptionGroup` / subcommand pattern. Each subcommand should live in `Sources/TerminalDashboard/Commands/`. The `auto` subcommand should be the `defaultSubcommand`. Shared CLI flags (e.g. `--config`) belong on the root command.

## Root cause

The entry point was a single flat `ParsableCommand` with no subcommands, leaving no structure for the five dashboard modes.

## Fix

Rewrote `TerminalDashboard.swift` as a root command with `CommandConfiguration` listing five subcommands and `defaultSubcommand: AutoCommand.self`. Created five files under `Sources/TerminalDashboard/Commands/`: `AutoCommand.swift`, `GitCommand.swift`, `CrashesCommand.swift`, `LogsCommand.swift`, `ConfigCommand.swift`. Each prints a "not yet implemented" placeholder. A `--config` `@Option` was added to the root command for future use.

## Verification

`./build.sh build` completes with "Build complete!" and no warnings. All five commands compile and the binary links successfully.
