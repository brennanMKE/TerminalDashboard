## Description

All three data sources emit events to the AutoCoordinator using a shared `DashboardEvent` type. This type and its `Severity` enum must be defined before any data source can be implemented.

## Expected behavior

`DashboardEvent.swift` defines:
- `enum Severity: Comparable { case info, warning, error, critical }`
- `struct DashboardEvent` with at minimum: `source` (which dashboard), `severity`, `message`, and `timestamp`.

## Actual behavior

No such type exists.

## Notes

`Severity` needs to be `Comparable` so the coordinator can determine highest-priority event across sources. The `source` field should be an enum of the three views (`git`, `crashes`, `logs`) so the coordinator can route without stringly-typed comparisons.

## Root cause

`DashboardEvent.swift` and the `DataSources/` directory did not exist; no shared event type had been defined for data sources to use.

## Fix

Created `Sources/TerminalDashboard/DataSources/DashboardEvent.swift` with:
- `enum Severity: Comparable, Sendable` — cases `info`, `warning`, `error`, `critical` in ascending order so Swift's synthesised `Comparable` implementation works naturally via raw case order.
- `enum DashboardSource: Sendable` — cases `git`, `crashes`, `logs`.
- `struct DashboardEvent: Sendable` — fields `source`, `severity`, `message`, and `timestamp`.
- `import Foundation` added to bring `Date` into scope.

## Verification

`./build.sh build` completes with "Build complete!" and zero errors or warnings.
