# 0005 — Define DashboardEvent type with severity levels

| | |
|---|---|
| **Status** | open |
| **Module** | DataSources |
| **Platform** | macOS |
| **First seen** | 2026-05-22 |

## Description

All three data sources emit events to the AutoCoordinator using a shared `DashboardEvent` type. This type and its `Severity` enum must be defined before any data source can be implemented.

## Expected behavior

`DashboardEvent.swift` defines:
- `enum Severity: Comparable { case info, warning, error, critical }`
- `struct DashboardEvent` with at minimum: `source` (which dashboard), `severity`, `message`, and `timestamp`.

## Actual behavior

No such type exists.

## Notes

`Severity` needs to be `Comparable` so the coordinator can determine highest-priority event across sources. The `source` field should be an enum of the three views (`git`, `crashes`, `logs`) so the coordinator can route without stringly-typed comparisons.
