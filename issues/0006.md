## Description

`GitDataSource` must poll the configured repository every 2 seconds using `git status --porcelain=v2 --branch`, parse the output into a typed model, and emit `DashboardEvent` values when significant changes occur.

## Expected behavior

- Parses ahead/behind counts and per-file status flags (M, A, D, ?, R) from porcelain v2 output.
- Emits `warning` when the branch falls behind remote.
- Emits `info` on any other working-tree change.
- Handles missing `git` binary or non-git directory gracefully — surfaces an error state without crashing.
- Exposes a `@Published` state value consumed by `GitView`.

## Actual behavior

No data source exists.

## Notes

Run `git` as a subprocess via `Process`. Parse `# branch.ab +N -N` for ahead/behind. Each changed file line begins with `1`, `2`, `?`, or `u` — see `git status --porcelain=v2` docs for the format. The data source is a Swift `actor` to ensure thread-safe state. It should accept an `AsyncStream` or `AsyncSequence` sink for events so the coordinator can subscribe.

## Root cause

No `GitDataSource` type existed in the `DataSources` module.

## Fix

Created `Sources/TerminalDashboard/DataSources/GitDataSource.swift` containing:

- `GitDataSourceError` — a typed `Error` enum covering all failure modes (git not found, launch failure, not-a-repository, non-zero exit).
- `GitFileStatus` — `Sendable` model struct with `path`, `stagedFlag`, and `worktreeFlag`.
- `GitDataSource` — Swift `actor` that polls `git status --porcelain=v2 --branch` every 2 seconds via `Process`, parses porcelain v2 headers and file-change lines, emits `DashboardEvent` values through an `AsyncStream` continuation (warning on behind 0→N, info on file-count change), and exposes read-only actor-isolated state.
- `GitState` — `@MainActor final class ObservableObject` that owns a `GitDataSource`, starts its polling loop, subscribes to the event stream, and mirrors state onto `@Published` properties for SwiftTUI views. Exposes an `onEvent` callback for coordinator integration.

Swift 6 strict concurrency is satisfied: `Result<String, String>` was replaced with `Result<String, GitDataSourceError>` (String does not conform to Error in Swift 6), all cross-actor state access uses `await`, and `Sendable` conformances are explicit.
