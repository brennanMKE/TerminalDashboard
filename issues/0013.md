## Description

Pressing Cmd+H from any view opens a modal overlay showing all available shortcuts. The overlay covers the content area while the tab bar (in auto mode) remains visible. Any keypress dismisses it.

## Expected behavior

**Navigation section** (auto mode only):
- Cmd+Shift+G — Switch to Git Status
- Cmd+Shift+C — Switch to Crashes
- Cmd+Shift+L — Switch to Logs
- Cmd+Shift+A — Toggle Auto Mode
- Cmd+H — Toggle this help screen
- q / Ctrl-C — Quit

In dedicated mode the navigation section is replaced by: "Running in dedicated mode — view switching disabled."

**Current view actions section** (dynamically populated):

Git Status: `p` Push, `u` Pull, `r` Rebase, `o` Open in external tool
Crashes: `↑`/`↓` Move selection, `e` Extract selected, `E` Extract all
Logs: `Space` Pause/resume, `c` Clear buffer

## Actual behavior

No overlay exists.

## Notes

The overlay is a SwiftTUI view rendered on top of the active content view. Confirm SwiftTUI supports z-axis overlays (`.overlay()` or `ZStack`) at implementation time. The key bindings shown should reflect the actual receivable shortcuts confirmed during implementation of view switching (see issue #0015) — update the overlay content if Cmd+Shift variants are not receivable by the app.

## Root cause

No `HelpOverlay` view existed; the feature was not yet implemented.

## Fix

Created `Sources/TerminalDashboard/Views/HelpOverlay.swift` as a standalone SwiftTUI `View`. The view accepts `isDedicatedMode: Bool`, `activeView: DashboardSource`, and `onDismiss: () -> Void`. It uses a `VStack` layout with:

- Bold "Keyboard Shortcuts" title and a `Divider`
- In auto mode: a Navigation section (header in bold cyan, six shortcut rows with a "(terminal permitting)" note on the header) followed by a `Divider`
- In dedicated mode: a gray "Running in dedicated mode — view switching disabled" note followed by a `Divider`
- A "Current View: \<name\>" section dynamically populated per `activeView` (Git / Crashes / Logs shortcuts)
- A `TextField` dismiss row: any input + Return calls `onDismiss()`

ZStack overlay support was confirmed present in SwiftTUI; however, the view is designed as a self-contained component that the parent can place inside a `ZStack` or swap in conditionally.
